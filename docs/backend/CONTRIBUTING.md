# Вклад в backend Universal Deep Research

Проект — исследовательский прототип. Любые изменения должны сохранять демонстрационный характер.

## Как участвовать
1. Сделайте форк и клонируйте репозиторий:
   ```bash
   git clone <ваш-fork-url>
   cd backend
   ```
2. Настройте окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
3. Вносите изменения, соблюдая существующий стиль. Добавляйте тесты и обновляйте документацию при необходимости.
4. Проверьте работу:
   - Протестируйте конечные точки API
   - Убедитесь, что конфигурация и переменные окружения корректны
5. Оформите PR с понятным названием и описанием изменений.

## Стандарты кода
- Следуйте PEP 8
- Пишите понятные docstring и комментарии
- Обрабатывайте ошибки явно
- Используйте переменные окружения для настройки
- Логи с осмысленными уровнями и сообщениями

## Лицензия
Вклад принимается на условиях проекта (только исследовательское/демо‑использование). См. [LICENSE](../LICENSE.txt).

## Подпись коммитов
Коммиты должны содержать `Signed-off-by`:
```bash
git commit -s -m "Add feature"
```
Тем самым вы подтверждаете право на внесение изменений согласно DCO 1.1.
